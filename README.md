# Progetto UNIBA/Laboratorio di Informatica di gruppo
# FabbAmp, perennemente in pre-alpha
Progetto universitario di gruppo intento alla creazione di un gestore dati per la musica, es: WinAmp (in versione un po' barbona), da realizzare nell'ambito del corso di studi di Laboratorio di informatica, a.a. 2019/20.

## Come funziona
Obiettivo del progetto è la gestione di una base dati musicale, al fine di fornire agli appassionati uno strumento per la ricerca di informazioni di qualunque genere riguardanti i brani e gli artisti di loro gradimento.

## Specifiche e funzionalità
### UX/UI:
Linea di comando

### Funzionalità del sistema:
Le operazioni che il sistema fornirà agli utenti si suddividono in X classi:
* Operazioni essenziali per la gestione dei dati
* Operazioni essenziali per la ricerca di informazioni
* Operazioni per la condivisione di informazioni
* Operazioni per l'ascolto dei brani musicali

### Gestione dei dati:
Le operazioni per la gestione dei dati sono fondamentali per qualunque base di dati. Sono le operazioni classiche che si eseguono su ogni base di dati:
* [Inserimento dei dati](https://github.com/Soxasora/FabbAmp/issues/5): Inserimento nella base dati di tutte le informazioni previste per un brano musicale
* [Modifica dei dati](https://github.com/Soxasora/FabbAmp/issues/6): modifica di una o più informazioni di un determinato brano musicale
* [Cancellazione dei dati](https://github.com/Soxasora/FabbAmp/issues/7): eliminazione di un brano musicale dalla base dati (con esso, tutte le informazioni)
* [Backup dei dati](https://github.com/Soxasora/FabbAmp/issues/8): copia periodica della base dati
* [Stampa di report](https://github.com/Soxasora/FabbAmp/issues/9): stampa di tutte o alcune delle informazioni relative a uno o più brani musicali selezionati

### Ricerca di informazioni:
Le operazioni di ricerca sono le operazioni più eseguite su ogni base di dati e, pertanto, devono essere efficienti e permettere agli utenti di cercare qualunque informazione sia di loro interesse. Nel caso del sistema **FabbAmp** saranno presenti le seguenti operazioni di ricerca:
* [Ricerca di brani in base a qualunque dato *anagrafico*](https://github.com/Soxasora/FabbAmp/issues/10): autore, titolo ecc.
* [Ricerca di brani in base al loro gradimento](https://github.com/Soxasora/FabbAmp/issues/11) tra gli utenti del sistema

### Condivisione di informazioni
Queste operazioni, così come le prossime, sono operazioni non necessarie ma che potrebbero risultare sicuramente gradite agli utenti del sistema. L'obiettivo di tali operazioni è quello di permettere la condivisione, con altri utenti, dei propri brani e/o playlist:
* [Condivisione di un brano](https://github.com/Soxasora/FabbAmp/issues/12) con uno o più utenti del sistema
* [Condivisione di una playlist](https://github.com/Soxasora/FabbAmp/issues/13) con uno o più utenti del sistema
* [Creazione di gruppi di utenti](https://github.com/Soxasora/FabbAmp/issues/14)

### Ascolto di brani
In questa categoria rientrano tutte le operazioni di ascolto/fruizione dei brani musicali.
* [Ascolto di un singolo brano](https://github.com/Soxasora/FabbAmp/issues/15)
* [Ascolto di una intera playlist](https://github.com/Soxasora/FabbAmp/issues/16)
* [Creazione di una playlist](https://github.com/Soxasora/FabbAmp/issues/17) personale

### Dati del sistema
I dati che saranno gestiti, per ogni   brano musicale saranno:
* Titolo del brano musicale
* Autore del brano musicale
* Anno di incisione del brano musicale
* LP/CD in cui è contenuto
